# План реализации: Телеграм-автопостер с управлением в Telegram и настройками на Сайте

План описывает пошаговые изменения для реализации мультиканального автопостинга (публичный/приватный канал), интерактивного управления очередью/предложкой прямо в мессенджере и удобной страницы настроек на веб-панели.

---

## 1. Изменения в Базе Данных (Схема Supabase)

Необходимо выполнить следующие миграции (добавить колонки) в существующие таблицы:

1. **Таблица `autopost_bots`:**
   * Добавить массив `admin_tg_ids bigint[]` (для поддержки нескольких администраторов).
   * Добавить JSONB-поле `active_modes` (для сохранения выбранного канала каждым админом, по умолчанию `{}`).
2. **Таблица `channels`:**
   * Добавить логическое поле `auto_accept_suggestions` (Boolean, по умолчанию `false`).
   * Добавить JSONB-поле `buttons_config` (массив объектов типа `[{"text": "Купить", "url": "https://..."}]`).
3. **Таблица `autopost_items`:**
   * Добавить массив `file_ids text[]` (вместо/в дополнение к `file_id` для поддержки альбомов).
   * Добавить `target_channel_id bigint` (ID конкретного канала, куда слать контент).

## 2. Предлагаемые изменения по компонентам

### Backend: Сервис автопостинга [autopost.service.js](file:///Users/webzwezda/Desktop/bullrun/backend/services/autopost.service.js)

1. **Исправление перезапуска (`server.js` + `service`):**
   * Заменить `startWebhookBot` на `startBot` (с запуском Polling через `bot.launch()`), чтобы боты просыпались после перезапуска сервера.
2. **Регистрация обработчиков Telegraf (в методе `registerHandlers`):**
   * **Команда `/start`:**
     * Если массив `admin_tg_ids` пуст -> выводить инлайн-кнопку «✅ Я администратор». При нажатии записывать ID в базу.
     * Если пишет админ (ID входит в массив) -> выводить клавиатуру админа (кнопки: Режим, Очередь, Добавить, Предложки).
     * Если пишет гость (переход по дип-линку `?start=suggest_pub` или `suggest_priv`) -> запоминать его контекст предложки и просить прислать фото.
   * **Клик по кнопке «🔄 Режим»:**
     * Переключать в БД активный канал админа (`active_modes`) между публичным и приватным.
   * **Прием фото и альбомов (событие `photo`):**
     * Добавить debounce-таймер (1 секунда) для склейки картинок с общим `media_group_id`.
     * Если прислан альбом -> присылать вопрос: «Склеить в альбом» или «Разбить».
   * **Действие «📋 Очередь»:**
     * Бот спрашивает, какую очередь показать. Выдает ленту постов с инлайн-кнопками управления под каждым сообщением:
       * `[ ⚡️ Опубликовать ]` — сразу шлет в канал и стирает сообщение.
       * `[ 📝 Изменить текст ]` — включает режим ожидания текста для конкретного поста.
       * `[ 🔒 В Приват ]` (или `[ 📢 В Паблик ]`) — меняет `target_channel_id` в БД, пост исчезает из текущего списка.
       * `[ ❌ Удалить ]` — удаляет из БД и стирает сообщение в чате.
   * **Авто-решедулинг (схлопывание очереди):**
     * При удалении или публикации поста вне очереди автоматически сбрасывать все будущие запланированные посты (`scheduled_at = null`, `status = 'queued'`) и вызывать заново `scheduleNextBatch()`, чтобы убрать «дыры».

### Backend: Маршрутизатор [autopost.routes.js](file:///Users/webzwezda/Desktop/bullrun/backend/routes/autopost.routes.js)

1. Добавить эндпоинты для управления списком администраторов (`GET /bots/:id/admins`, `POST /bots/:id/admins`, `DELETE /bots/:id/admins/:tgId`).
2. Добавить/обновить роут `PATCH /bots/:id/channels/:channelId` для сохранения индивидуальных настроек канала: частота постинга (`posts_per_day`), автопринятие (`auto_accept_suggestions`) и инлайн-кнопки (`buttons_config`).

### Frontend: Панель настроек [QuickStartPage.jsx](file:///Users/webzwezda/Desktop/bullrun/admin-v2/src/pages/QuickStartPage.jsx)

1. **Мастер онбординга:**
   * Скрыть все настройки, пока бот не подключен.
   * Если бот подключен, но `admin_tg_ids` пуст -> показывать карточку ожидания: *«Откройте @username и нажмите /start»*.
   * Если админ привязан, но каналов нет -> показывать карточку ожидания: *«Сделайте бота админом в ваших каналах»*.
2. **Панель настройки каналов (после прохождения онбординга):**
   * Разделить настройки каналов на вкладки (Tabs) для открытого и закрытого каналов.
   * Во вкладке каждого канала разместить:
     * Выбор частоты постов (1-3 в день).
     * Тумблер (Switch) автопринятия предложений (`auto_accept_suggestions`).
     * **Конструктор кнопок под постами:** динамическая форма, позволяющая добавлять строки `Текст кнопки` и `Ссылка`, с возможностью удалять строки и добавлять новые.
3. **Блок управления админами (В самом низу):**
   * Список текущих админов с возможностью удалить любого вторичного админа.
   * Инпут для ручного добавления Telegram ID.
   * Кнопка для генерации инвайт-ссылки.

## 3. План верификации

### Ручное тестирование в Telegram:
1. Подключить тестового бота, пройти процедуру привязки владельца через `/start` и кнопку «Я администратор».
2. Добавить бота в тестовый публичный и приватный каналы. Проверить автодетект на странице сайта.
3. Сменить направление на «Публичный» и закинуть альбом из 3 картинок. Выбрать «Разбить» и «Опубликовать альбомом» для проверки работы альбомов.
4. Проверить кнопку «Очередь», изменить текст подписи к одному из постов, удалить один пост и опубликовать другой досрочно. Убедиться, что сообщения исчезают/меняются и очередь автоматически сдвигается вперед без задержек.
5. Написать боту с другого (гостевого) аккаунта, прислать мем в предложку. Зайти под аккаунтом админа, нажать кнопку «Предложки», протестировать публикацию предложки вне очереди и перенос в общую очередь.

### Проверка работоспособности кода:
1. Запустить линтинг проекта и проверить сборку фронтенда:
   ```bash
   cd admin-v2 && npm run build
   ```
2. Проверить, что бекенд корректно стартует и инициализирует ботов:
   ```bash
   cd backend && node server.js
   ```
