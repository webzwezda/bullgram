Этот пост создан специально для того, чтобы показать, как выглядят различные элементы Markdown. По умолчанию Eleventy использует парсер `markdown-it`, который поддерживает стандарты CommonMark, а также некоторые популярные расширения.

## 1. Заголовки (Headings)

Вот примеры заголовков разных уровней. Старайтесь не использовать `H1` внутри поста, так как он обычно резервируется для главного заголовка страницы (в данном случае "Демонстрация элементов Markdown").

### Заголовок уровня 3 (H3)

#### Заголовок уровня 4 (H4)

##### Заголовок уровня 5 (H5)

## 2. Текст и форматирование (Text Formatting)

В тексте можно выделять важные фрагменты. Например, **жирный текст**, *курсив*, ***жирный курсив*** или даже ~~зачеркнутый текст~~.

Также часто используется выделение `inline-кода`, когда нужно упомянуть название переменной или небольшую команду в тексте.

## 3. Ссылки и изображения (Links and Images)

Обычная текстовая ссылка выглядит так: [Официальный сайт Eleventy](https://www.11ty.dev/).
Ссылки могут быть с `title` атрибутом: [Наведите на меня](https://google.com "Привет! Это всплывающая подсказка").

Пример изображения:
![Пейзаж (заглушка)](https://picsum.photos/800/400 "Случайное изображение с picsum")

## 4. Списки (Lists)

### Маркированный (Unordered)

- Первый пункт списка
- Второй пункт списка
  - Вложенный пункт 2.1
  - Вложенный пункт 2.2
    - Еще глубже
- Третий пункт списка

### Нумерованный (Ordered)

1. Первый шаг
2. Второй шаг
   1. Подшаг 2.1
   2. Подшаг 2.2
3. Третий шаг

## 5. Блочные цитаты (Blockquotes)

Блочные цитаты используются для выделения мыслей, правил или прямой речи.

> Это пример блочной цитаты.
> Она может занимать несколько строк.
> 
> И даже содержать другие элементы, например **жирный текст** или списки:
> - Пункт внутри цитаты.

## 6. Блоки кода (Code Blocks)

Для многострочного кода используйте тройные обратные кавычки. Парсер попытается подсветить синтаксис (если подключены соответствующие плагины/стили, вроде Prism.js).

```js
// Пример JavaScript функции
function greetUser(name) {
  if (!name) {
    console.log("Привет, аноним!");
    return;
  }
  console.log(`Привет, ${name}! Добро пожаловать в BullRun.`);
}

greetUser("Админ");
```

```css
/* Пример CSS */
.demo-box {
  background-color: var(--blue-soft);
  border: 1px solid var(--blue);
  border-radius: 8px;
  padding: 16px;
}
```

## 7. Таблицы (Tables)

Таблицы отлично подходят для структурирования данных, например, тарифов или сравнений.

| Название тарифа | Цена (TON) | Срок | Описание |
| :--- | :---: | :---: | ---: |
| **Start** | 5 | 1 месяц | Базовый доступ в канал |
| **Pro** | 12 | 3 месяца | Канал + закрытый чат |
| **VIP** | 40 | 1 год | Все привилегии + личные звонки |

*(Обратите внимание на выравнивание столбцов: слева, по центру, справа).*
